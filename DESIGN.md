# Design Context

## Direction

Use a light, technical lab-note style: crisp typography, visible structure, restrained color, and an explicit payment flow diagram. The page should feel like something a protocol team would proudly link from docs.

## Palette

Use OKLCH colors. Prefer tinted off-white surfaces, graphite text, green for successful transfer/action, and amber only for testnet warnings. Avoid dark blue gradients and generic crypto neon.

## Components

- Top navigation with token and environment metadata.
- First viewport split: CCH narrative and faucet action.
- Form controls use standard browser affordances with strong focus states.
- Status messages are inline, not modal.
- Flow diagram uses labeled nodes and lines, not decorative icons.

## Copy Rules

English only. Be direct. Do not use em dashes. Avoid generic phrases such as "seamless", "unlock", "revolutionary", or "next generation".
